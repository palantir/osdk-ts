# ObjectTable

An OSDK-aware data table for ontology objects. Renders rows from an
`ObjectType`, `ObjectSet`, or interface, and ships with column configuration,
selection, sorting, pinning, resizing, inline editing, and an integrated
filter surface.

<Source
  language="tsx"
  code={`import { ObjectTable } from "@osdk/react-components/experimental/object-table";`}
/>

## Usage

Pass an OSDK object type. ObjectTable will fetch rows, infer columns, and
wire up sort / filter / selection out of the box.

<Source
  language="tsx"
  code={`import { Employee } from "@my/osdk";
import { ObjectTable } from "@osdk/react-components/experimental/object-table";

export function EmployeesTable() {
  return <ObjectTable objectType={Employee} />;
}`}
/>

### Prerequisites

- Wrap your app in `OsdkProvider` (see [Prerequisites](?path=/docs/prerequisites--docs)).
- Import the package CSS and configure `@layer` order — see the

### Data Attributes

Stateful attributes you can target with CSS selectors. Prefer these over CSS
class names — class names are CSS-module-hashed and not stable.

| Attribute | Element | Values | When applied |
|---|---|---|---|
| `data-resizing` | `<thead>` | `"true"` &#124; `"false"` | Set while the user is dragging a column resizer |
| `data-pinned` | `<th>`, `<td>` | `"left"` &#124; `"right"` &#124; `"false"` | Reflects each column's pin state |
| `data-selected` | `<tr>` | `"true"` &#124; `"false"` | Set on selected rows |
| `data-focused` | `<tr>` | `"true"` &#124; `"false"` | Set on the row currently focused by keyboard navigation |
| `data-row-parity` | `<tr>` | `"even"` &#124; `"odd"` | Row striping; useful for zebra patterns at the consumer level |
| `data-editable` | `<td>` | `"true"` &#124; (absent) | Set when a cell can be edited in the current `editMode` |

### Selectors

ObjectTable uses CSS Modules internally, so class names are hashed at build
time and **not** part of the public API. To style states or variants from a
consuming app, target the documented data attributes instead.

<Source
  language="css"
  code={`/* Highlight selected rows with a custom accent */
[data-selected="true"] {
  background: var(--my-app-selected-row-bg);
}

/* Style the divider between pinned and unpinned columns */
[data-pinned="left"] + [data-pinned="false"] {
  border-left: 2px solid var(--my-app-pinned-divider);
}

/* Visually mark editable cells */
[data-editable="true"] {
  background: var(--my-app-editable-tint);
}`}
/>

## Accessibility

- The table renders as a native `<table>` with `<thead>` / `<tbody>` / `<tr>` / `<th>` / `<td>` for screen reader support.
- Row focus is exposed via `data-focused` so keyboard navigation has a stable styling hook.
- Editable cells use focusable form controls from `@base-ui/react`; tab order follows visual order.
- When customizing colors via CSS variables, verify contrast against the active theme (light + dark).
