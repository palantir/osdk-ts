# Plan: FilterList — right-align "Clear all" in the exclude row

## Why
In the exclude row (`FilterInputExcludeRow`), the layout is:

```
[ ExcludeDropdown ] [ count label ] [ Clear all ]
```

inside a `.excludeRow` flex container. The intent is for "Clear all" to sit
flush against the right edge so the count label and Clear All read as two
separate affordances. Today they visually smush together on the left because
the flex container collapses to its natural content width, leaving
`margin-left: auto` (already declared on `.clearAllButton`) with no slack to
push against.

Inspection of `FilterListItem.module.css` confirms:
- `.clearAllButton { margin-left: auto; ... }` — already set
- `.excludeRow { display: flex; align-items: center; ... }` — present, but
  **`width: 100%` is missing**, so the row only stretches to its content

Adding `width: 100%` (and confirming `margin-left: auto` on the button) makes
the auto-margin push Clear All to the right edge as intended. No JSX change
is required.

## Approach
CSS-only fix scoped to `FilterListItem.module.css`:

1. Add `width: 100%` to `.excludeRow` so the flex container fills the parent
   horizontally and `margin-left: auto` on `.clearAllButton` has room to
   expand.
2. Verify `.clearAllButton { margin-left: auto; }` (already present) — keep
   the rule explicit.
3. No changes to `FilterInputExcludeRow.tsx`.
4. Manually verify in the storybook story for the listogram / multi-select /
   single-select filter (exclude row open, with selections so Clear All
   renders) that the button docks to the right edge across viewport widths.
